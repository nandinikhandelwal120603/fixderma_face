B2. WHAT WE ARE BUILDING
An AI-powered Visual Skin Diagnostic and Product Recommendation Engine that does
the following in one seamless flow:
User clicks/uploads a photo of their skin concern → AI identifies the exact condition →
Explains what it is and why it occurs → Recommends the precise Fixderma product(s)
from their live catalog → Tells the user how to use the product, what results to expect,
and realistic timelines → Provides a direct “Buy Now” link to fixderma.com.
This can be deployed across multiple surfaces (discuss priority with client): a web widget
embeddable on fixderma.com (replacing/upgrading their current Face Analysis page), a
feature within their existing mobile app, and/or a WhatsApp bot for maximum reach in
India. Our recommendation is to build the web widget first (fastest to deploy, works on all
devices) and the WhatsApp flow second (highest reach in Tier 2/3 India).
B3. THE USER JOURNEY — DETAILED FLOW
Step 1 — Capture: User opens the tool. They see a clean, clinical interface with a prominent
camera icon and the headline: “Show us your skin concern.” Two options: take a live photo
using the device camera, or upload an existing image from gallery. Guidance text appears:
“For best results: use natural lighting, hold the camera 6–8 inches from the area, avoid filters
or makeup on the affected area, ensure the image is focused and clear.” Allow the user to
crop/frame the specific area of concern before submitting.
Step 2 — AI Analysis (the brain): The image is sent to a vision AI model. The model
performs multi-label skin condition classification. It must be trained/prompted to identify the
following conditions:

Acne (with severity grading: mild — a few whiteheads/blackheads; moderate — papules and
pustules; severe — nodules and cysts). Hyperpigmentation and dark spots. Melasma
(specifically — hormonal pigmentation, typically butterfly pattern on cheeks). Acanthosis
Nigricans (darkened, velvety patches on neck, underarms, elbows, knuckles — this is critical
because Nigrifix is their hero product). Dark circles under eyes. Open/enlarged pores.
Dry/flaky skin (including xerosis). Oily/shiny skin. Scars (acne scars, post-inflammatory
marks, surgical scars). Stretch marks. Cracked heels. Signs of ageing (fine lines, wrinkles,
loss of elasticity, sagging). Sunburn / sun damage. Rosacea / redness / sensitive skin. Eczema
/ dermatitis (general identification). Vitiligo (white/depigmented patches). Dandruff / scalp
flaking (if scalp image submitted). Hair thinning / hair fall pattern (if scalp image submitted).
The model should output a structured response: primary condition identified (highest
confidence), secondary conditions (if any), confidence score for each, severity assessment
(mild/moderate/severe where applicable).
Step 3 — Condition Explanation (Educational Layer): Once identified, the app presents a
clear, jargon-free explanation organized into three sections:
“What is it” — A 2–3 sentence description of the condition in everyday language. Example
for Acanthosis Nigricans: “This is a skin condition where certain areas — often the neck,
underarms, or body folds — develop darker, thicker, velvety patches. It’s very common and
not harmful, but it can be a sign that your skin needs targeted care.”
“Why does it happen” — Common causes relevant to the condition. Example: “Acanthosis
Nigricans is often linked to insulin resistance, hormonal changes, obesity, or friction in skin
folds. It’s especially common in people with PCOS or prediabetic conditions.”
“How serious is it” — Either reassurance (“This is a cosmetic concern that responds well to
topical treatment”) or a flag to consult a dermatologist (“Based on what we see, we
recommend consulting a dermatologist for a clinical evaluation alongside using skincare
products”).
Step 4 — Product Recommendation (The Conversion Engine): Based on the identified
condition, the system maps to the exact Fixderma product(s). This mapping is NOT generated
freely by the LLM — it is driven by a structured Condition → Product Mapping Table that
we build and maintain from Fixderma’s catalog data. The LLM identifies the condition; a
deterministic lookup returns the products. This prevents hallucinated product names or
incorrect recommendations.

B4. TECHNICAL DIRECTION
Vision AI Model: Use GPT-4o Vision API or Claude Vision API as the primary analysis
engine. These multimodal models are excellent at skin condition classification when given

precise prompts. The system prompt must instruct the model to output a structured JSON: {
&quot;primary_condition&quot;: &quot;acanthosis_nigricans&quot;, &quot;confidence&quot;: 0.87,
&quot;severity&quot;: &quot;moderate&quot;, &quot;secondary_conditions&quot;: [{&quot;condition&quot;: &quot;dry_skin&quot;,
&quot;confidence&quot;: 0.45}], &quot;recommendation_ids&quot;: [&quot;nigrifix_cream_100g&quot;,
&quot;nigrifix_roll_on&quot;] }. The recommendation_ids are looked up from our product
database — the LLM does NOT generate product names freely.
Alternative (cost-optimized): Fine-tune an open-source vision model (e.g., LLaVA, or a
dermatology-specific model from Hugging Face like dermnet-classifier) for the classification
step, and use a smaller LLM (GPT-4o-mini or Claude Haiku) for generating the explanation
text. This reduces per-analysis API cost from ~$0.03–0.05 to under $0.01.
Product Catalog Backend: Build a structured product database. Sources: scrape from
fixderma.com or manually compile from their site. Schema per product: product_id (our
internal ID), name, fixderma_url, image_url, mrp, sale_price, active_ingredients (array),
skin_concerns_addressed (array of condition tags), how_to_use (text),
expected_results_timeline (text), category, sub_category. Store in Supabase or Airtable
(Airtable is easier for the client to update themselves via spreadsheet-like UI). This database
is the single source of truth — the AI recommendation is only as good as this data. Build an
admin interface for Fixderma’s team to add new products, update prices, and modify concern-
to-product mappings without developer involvement.
Recommendation Logic: Two-layer architecture. Layer 1: Vision AI classifies the skin
condition. Layer 2: A deterministic rules engine maps condition → product_ids using the
mapping table. The LLM’s classification output feeds into the rules engine. The rules engine
queries the product database and returns the matched products. This ensures no hallucinated
recommendations.
Front-End: Build as a responsive PWA using Next.js or a Replit/Bolt-generated interface.
Mobile-first design. Clean, clinical aesthetic — white background, Fixderma’s teal/green
brand accents, clear typography, high-quality product images. Must load fast on Indian
mobile networks. The camera interface should use the device’s native camera API for best
quality.
WhatsApp Integration (high-value second surface): Use the WhatsApp Business API via
Gupshup (India-preferred) or Twilio. Flow: User sends a photo of their skin concern to the
Fixderma WhatsApp business number. The bot acknowledges: “Analyzing your skin… this
will take a moment.” Processes the image through the same vision AI pipeline. Replies with a
structured message: condition identified, brief explanation, top 2–3 product recommendations
with images and “Buy Now” links. This is extremely powerful because WhatsApp reach in
India is universal — far beyond app downloads.
Privacy &amp; Data Architecture: Skin images are highly sensitive personal data. The system
must NOT store images on any server beyond the duration of the analysis session unless the
user explicitly opts in (e.g., “Save my analysis to track progress over time”). If images are
stored with consent, they must be encrypted at rest. Comply with Indian data protection
regulations (DPDP Act 2023). Display a clear privacy policy before the first image capture.
No images should be used for model training without explicit separate consent.
B5. SUCCESS METRICS

Analysis accuracy: target above 85% agreement with dermatologist review on the primary
condition identified (validate with 100 test cases reviewed by a dermatologist). Add-to-cart
conversion rate from the recommendation page: target above 20%. Average session depth:
measure how many users read the full explanation, view all recommended products, and click
“Buy Now.” Repeat usage: do users come back for different skin concerns or to track
improvement over time — target 15%+ return rate within 30 days. Revenue attribution: track
how much GMV on fixderma.com is driven by the AI tool (UTM parameter on all “Buy
Now” links).
B6. WATCH-OUTS &amp; RISK MITIGATION
The AI must never “diagnose” — always frame output as “assessment,” “analysis,” or “our
AI suggests.” For conditions that appear clinically significant or beyond cosmetic skincare
(severe eczema, psoriasis, potential malignancy, deep ulceration), the system must refuse to
recommend products and instead display only a dermatologist consultation advisory. The
product mapping table must be maintained as Fixderma launches new products — this is an
ongoing operational task. Build automated alerts if a product URL returns a 404 (product
discontinued or renamed). Test the vision AI with diverse Indian skin tones — Fitzpatrick
types III through VI dominate the Indian market. The model must perform equally well
across all skin tones. Poor performance on darker skin tones is both an ethical failure and a
business failure (Nigrifix’s core user base has darker skin in the affected areas).
